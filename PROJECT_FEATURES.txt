CORE FEATURES:
==============

1. FACULTY-ONLY LOGIN (ROLE-BASED AUTHENTICATION)
   ├── Admin (Exam Cell) & Faculty login capability
   ├── Session-based authentication with username/password
   ├── Role detection: Admin vs Faculty
   ├── Role-based access control
   ├── Automatic redirection to appropriate dashboard
   ├── Logout functionality
   ├── Protected views with @login_required decorator
   ├── Admin (is_staff=True): Full system access
   ├── Faculty: Only their assigned subjects
   └── Students: NO access to system (by design)

2. SEMESTER-WISE STUDENT MANAGEMENT
   ├── Store students with: roll number, name, enrollment, branch, semester
   ├── Global search by roll number OR name OR enrollment number
   ├── Filter by branch AND semester
   ├── Add new student form (Admin only)
   ├── View all student details with profile page
   ├── Database model: Student with all required fields
   ├── Edit student information (Admin)
   ├── Delete student functionality (Admin)
   └── Upload student lists from Excel/CSV files

3. STUDENT PROFILE PAGE (KEY DIFFERENTIATOR)
   ├── COMPREHENSIVE ONE-PAGE STUDENT VIEW:
   │   ├── Basic Student Details
   │   │   ├── Roll number
   │   │   ├── Name
   │   │   ├── Enrollment number
   │   │   ├── Branch
   │   │   ├── Semester
   │   │   ├── Email
   │   │   └── Phone
   │   │
   │   ├── Attendance Summary (Subject-wise %)
   │   │   ├── For each subject: Present count
   │   │   ├── For each subject: Total classes
   │   │   ├── For each subject: Percentage calculation
   │   │   └── Status indicator (Good >= 75%, Low < 75%)
   │   │
   │   ├── Marks Summary (Internal + External)
   │   │   ├── For each subject: Internal marks
   │   │   ├── For each subject: External marks
   │   │   ├── For each subject: Total marks
   │   │   ├── For each subject: Max marks
   │   │   ├── Grade assignment (A+, A, B+, B, C, F)
   │   │   ├── Pass/Fail status
   │   │   └── Overall percentage calculation
   │   │
   │   └── Exam Seating Information (Read-Only)
   │       ├── Exam number
   │       ├── Room number
   │       └── Seat number
   │
   └── Design: Student-centric view (not spreadsheet), color-coded indicators

4. DAILY ATTENDANCE MARKING (FASTER THAN EXCEL)
   ├── Faculty selects subject + date
   ├── Student list loads automatically from StudentSubject enrollments
   ├── DEFAULT: All students marked as PRESENT
   ├── Faculty clicks ONLY to mark absent (key speed feature!)
   ├── One-click save with auto percentage calculation
   ├── Prevents duplicates (update_or_create logic)
   ├── Store marked_by faculty reference with timestamp
   ├── Context memory: Remembers last subject used
   ├── Auto-calculates: Present_count / Total_classes * 100
   ├── Toggle button (P/A) instead of dropdown selection
   ├── Single save operation
   ├── Status: P (Present) / A (Absent)
   └── Works for any number of students per subject

5. MARKS MANAGEMENT SYSTEM
   ├── Subject-wise marks entry by faculty
   ├── Auto total calculation: internal + external
   ├── Grade assignment logic:
   │   ├── A+ >= 90
   │   ├── A >= 80
   │   ├── B+ >= 70
   │   ├── B >= 60
   │   ├── C >= 50
   │   └── F < 50
   ├── Pass/fail logic: >= 40 passes, < 40 fails
   ├── Input validation: Max marks checking (prevents invalid entries)
   ├── Safe-edit protection: Lock after final submit
   ├── Two-stage submission:
   │   ├── Save (draft mode, can edit multiple times)
   │   └── Submit & Lock (final mode, cannot edit)
   ├── Stores submitted flag, submitted_by faculty, submitted_date timestamp
   ├── Separate fields: Internal and External marks
   ├── Real-time total calculation
   ├── Status column shows complete/pending marks
   └── Faculty can only submit once

6. AUTOMATED EXAM SEATING + DIRECT PDF PRINT
   ├── Smart room allocation algorithm:
   │   ├── Input: total students, students per room, max per room
   │   ├── Input: room numbers list
   │   ├── Auto room allocation with intelligent distribution
   │   ├── Handle leftover students intelligently
   │   ├── Step 1: Assign preferred per room to ALL rooms
   │   ├── Step 2: Distribute remaining students to LAST rooms first
   │   ├── Step 3: Verify all students allocated
   │   └── Result: Even distribution, no room overflow
   │
   ├── Print-ready PDF output:
   │   ├── Institution header with logo
   │   ├── Semester, month, year display
   │   ├── Date range information
   │   ├── Table with room allocation details
   │   ├── Seat ranges (seat from - seat to)
   │   ├── Student counts per room
   │   ├── Notes/comments support
   │   └── A4 format optimized for printing
   │
   ├── Exam seating records in database:
   │   ├── Room number assignment
   │   ├── Seat ranges per room
   │   └── Student count per room
   │
   └── Workflow:
       ├── Click "Exam Seating" → Fill allocation form
       ├── Click "Generate Seating" → Auto-allocates students
       ├── Review in preview page → Check allocation status
       └── Click "Download PDF" → Get print-ready file

7. SMART DASHBOARD (DUAL ROLE)
   ├── ADMIN DASHBOARD:
   │   ├── Overview statistics:
   │   │   ├── Total students count
   │   │   ├── Total subjects count
   │   │   ├── Total faculty count
   │   │   └── System status
   │   ├── Recent activities:
   │   │   ├── Recent marks submissions
   │   │   └── Recent attendance records
   │   ├── Quick access links:
   │   │   ├── Manage students
   │   │   ├── Manage subjects
   │   │   ├── Manage exam seating
   │   │   └── View all faculty
   │   └── System overview for exam cell coordination
   │
   └── FACULTY DASHBOARD:
       ├── Quick subject cards with subject info
       ├── Pending work indicators:
       │   ├── Attendance not marked today (with count)
       │   ├── Marks not submitted (with count)
       │   └── Visual badges and warnings
       ├── Context memory display:
       │   ├── Last accessed subject
       │   └── Last accessed semester
       ├── Quick action links:
       │   ├── Mark attendance
       │   ├── Enter marks
       │   ├── View student profiles
       │   └── View seating
       └── Subject-wise navigation

8. EXCEL/CSV UPLOAD FOR STUDENT LIST
   ├── Upload Excel (.xlsx) or CSV (.csv) files
   ├── File format requirements:
   │   ├── Column 1: Enrollment Number
   │   └── Column 2: Student Name
   ├── Automatically assign exam numbers (sequential)
   ├── Automatic student distribution across rooms
   ├── Configurable options:
   │   ├── Choose starting exam number
   │   ├── Set starting seat number
   │   ├── Enter total number of students
   │   ├── Set preferred students per room
   │   ├── Set maximum students per room
   │   ├── Enter room numbers (e.g., 301-305,307)
   │   ├── Semester, month, year, date range
   │   └── Optional notes
   ├── Generate seating arrangement
   ├── View student list with exam numbers and room assignments
   ├── Download Seating Arrangement PDF
   ├── Download Student List PDF
   └── Professional PDF output matching institution format

9. SYSTEM FEATURES & SECURITY
   ├── Authentication:
   │   ├── Username/Password login
   │   ├── Session-based authentication
   │   ├── CSRF protection on all forms
   │   └── @login_required decorator on protected views
   │
   ├── Authorization:
   │   ├── Admin-only views: student management, exam seating
   │   ├── Faculty-only views: attendance, marks
   │   ├── Student-only views: NONE (by design)
   │   └── Role checking in all views
   │
   ├── Data Protection:
   │   ├── Passwords hashed using Django defaults
   │   ├── SQL injection prevented (ORM usage)
   │   ├── Can't edit locked marks after submission
   │   ├── Attendance records timestamped
   │   └── Audit trail: marked_by, submitted_by fields
   │
   └── Smart Features:
       ├── Context Memory:
       │   ├── Faculty.last_subject
       │   ├── Faculty.last_semester
       │   └── Pre-population of forms
       │
       ├── Default Present Attendance:
       │   ├── All auto-marked present (can't forget)
       │   ├── Much faster than manual checking
       │   └── Click only to mark absent
       │
       ├── Auto Calculations:
       │   ├── Attendance percentage: Present/Total * 100
       │   ├── Marks total: Internal + External
       │   ├── Grade: Based on total marks
       │   ├── Pass/Fail: >= 40 or < 40
       │   └── Prevents manual calculation errors
       │
       └── Room Allocation Algorithm:
           ├── Smart distribution logic
           ├── Even distribution across rooms
           └── No room overflow

12. WORKFLOW DESCRIPTIONS
    ├── LOGIN WORKFLOW:
    │   ├── Enter credentials
    │   ├── System validates user
    │   ├── Redirect admin → admin_dashboard
    │   └── Redirect faculty → faculty_dashboard
    │
    ├── ATTENDANCE WORKFLOW:
    │   ├── Faculty selects subject
    │   ├── Faculty selects date
    │   ├── System loads all enrolled students as PRESENT
    │   ├── Faculty clicks absent button on absentees only
    │   ├── Faculty clicks "Save Attendance"
    │   └── System saves attendance and calculates percentage
    │
    ├── MARKS WORKFLOW:
    │   ├── Faculty selects subject
    │   ├── System loads all enrolled students
    │   ├── Faculty enters internal marks
    │   ├── Faculty enters external marks
    │   ├── System calculates total and grade
    │   ├── Faculty clicks "Save Marks" (draft mode)
    │   ├── Faculty can edit multiple times
    │   ├── Faculty clicks "Submit & Lock" (final mode)
    │   └── System locks all marks (cannot edit)
    │
    ├── EXAM SEATING WORKFLOW:
    │   ├── Admin fills allocation form (students, rooms, preferences)
    │   ├── System runs smart allocation algorithm
    │   ├── Admin reviews preview
    │   ├── Admin downloads PDF
    │   └── Admin prints or shares
    │
    └── STUDENT PROFILE WORKFLOW:
        ├── Any user searches student
        ├── System loads student profile page
        ├── One page shows: details, attendance, marks, seating
        └── Easy to understand student status

13. DESIGN PHILOSOPHY
    ├── Faculty-First:
    │   ├── Designed around faculty workflow
    │   ├── Minimum clicks to mark attendance
    │   ├── Auto calculations reduce manual work
    │   ├── Context memory for quick access
    │   └── Pending work indicators
    │
    ├── Student-Centric:
    │   ├── Profile page focused on student data
    │   ├── Not spreadsheet-centric
    │   ├── One place to see: attendance, marks, grades, seating
    │   └── Easy to understand status with indicators
    │
    ├── Mistake Prevention:
    │   ├── Default present attendance (can't miss marking)
    │   ├── Input validation (max marks)
    │   ├── Lock after submit (prevent accidental changes)
    │   └── Confirmation on critical actions
    │
    └── Minimal & Clean:
        ├── No over-engineering
        ├── Simple HTML + CSS design
        ├── Readable code
        ├── No unnecessary features
        └── No AI/ML/Analytics
